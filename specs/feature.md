# specs/feature.md
# Contrato de features — mantém agentes honestos sobre o que foi implementado.
# Descreve o que cada feature deve fazer, não como está implementada.
# Use como referência de aceitação antes de marcar uma feature como concluída.

## Formato

```
### [nome-da-feature]
- [ ] Comportamento esperado 1
- [ ] Comportamento esperado 2
- [ ] Casos de erro tratados
```
